# T05 — Port `artifact-upload` module to mobile-automated + wire into bootstrap-orchestrator

**Status**: pending (orchestrator wired with stub; module logic pending port)
**Repo**: `takamulai/mithril` — `iac/cicd-onprem/pipelines/mobile-automated/modules/artifact-upload.Jenkinsfile`
**Blocks**: T06 (MDM step needs the Nexus URL from this stage)
**Blocked by**: T03

## Reason

The bootstrap pipeline needs to publish the built APK to Nexus so it can be
pulled by downstream distribution (MDM, manual install QR). The legacy
module at
`iac/cicd-onprem/pipelines/mobile/modules/artifact-upload.Jenkinsfile`
already does this — reads the APK out of the controller archive (see
[[FINDING §F01]], [[FINDING §F03]]) and `curl`s it to a Nexus raw repo.

Port verbatim with mobile-automated path tweaks. The module **must continue to
run on the Jenkins controller** ([[FINDING §F02]]).

## Input

- Read `iac/cicd-onprem/pipelines/mobile/modules/artifact-upload.Jenkinsfile`
  in full — including the comment block at lines 75-101 that documents
  the missing-plugin workaround.
- Confirm the Nexus URL `https://dsobinaryrepo.cbuae.gov/repository/retail-cbdc-artifacts`
  and credential `Nexus-cred` from the legacy module — reused as-is.

## Output

### `iac/cicd-onprem/pipelines/mobile-automated/modules/artifact-upload.Jenkinsfile`

Port the legacy module with these changes:

- Header references `mobile-automated/` and this T05 spec; cites [[FINDING §F01]]
  for the no-plugin context.
- `agent { label 'Jenkins' }` — keep, do not change (see [[FINDING §F02]]).
- Same parameter set: `ENVIRONMENT`, `ARTIFACT_TYPE` (`apk|aab|ipa`),
  `BUILD_TAG`, `SOURCE_JOB`, `SOURCE_BUILD_NUMBER`.
- Path translation `replace('/', '/jobs/')` left as-is — see
  [[FINDING §F04]].
- After successful upload, the module **must** echo the Nexus URL of
  the uploaded artefact, and (if possible) expose it via
  `buildVariables` as `NEXUS_ARTIFACT_URL`, so the MDM-distribute
  stage ([[T06]]) can pick it up from `result.buildVariables`.

### Bootstrap orchestrator wiring (already done; this spec documents contract)

`mobile-automated/orchestrators/bootstrap-orchestrator.Jenkinsfile`:

- `Upload Android Artifact` stage gated by
  `when { expression { params.ENABLE_UPLOAD && params.OS in ['Android','Both'] && env.ANDROID_BUILD_STATUS == 'SUCCESS' } }`.
- Calls `build job: 'Retail-CBDC/00-Mobile/Modules/artifact-upload'`
  with `ENVIRONMENT=${env.TARGET_ENV}`, `ARTIFACT_TYPE=apk`,
  `BUILD_TAG=${env.BUILD_TAG}`, `SOURCE_JOB=${env.ANDROID_BUILD_JOB}`,
  `SOURCE_BUILD_NUMBER=${env.ANDROID_BUILD_NUMBER}`.
- Captures `result.buildVariables.NEXUS_ARTIFACT_URL` into
  `env.NEXUS_ARTIFACT_URL` for [[T06]].

## Acceptance criteria

- Running the bootstrap-orchestrator with `ENABLE_UPLOAD=true` (default) and a successful
  Android build uploads the APK to
  `https://dsobinaryrepo.cbuae.gov/repository/retail-cbdc-artifacts/MobileApp/Android/<UTC-timestamp>/<filename>.apk`.
- Module emits `NEXUS_ARTIFACT_URL` in `buildVariables` and the
  orchestrator surfaces it in the post-summary.
- Running with `ENABLE_UPLOAD=false` skips the upload stage cleanly
  (visible as `Skipped` in Jenkins UI).
- If the build was discarded (`numToKeepStr` rolled over the build
  number), upload fails fast with the legacy module's exact error
  message — "Build archive not found: …" — see [[FINDING §F03]].

## Notes

- The Nexus path includes a UTC timestamp directory — multiple uploads
  of the same `BUILD_TAG` don't collide. Don't try to "deduplicate" by
  removing the timestamp; that breaks audit trail.
- The credential `Nexus-cred` is shared with the backend `docker-push`
  module. Don't rotate it without coordinating with the backend team.
