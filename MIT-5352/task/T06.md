# T06 ‚Äî Create `mdm-distribute` module (Nexus ‚Üí MDM) + wire into SIT

**Status**: pending (echo-only placeholder lands now; real MDM wiring blocked by MDM platform)
**Repo**: `takamulai/mithril` ‚Äî `iac/cicd-onprem/pipelines/mobile2/modules/mdm-distribute.Jenkinsfile`
**Blocks**: ‚Äî
**Blocked by**: T05 (MDM step consumes the Nexus URL produced by upload)

## Reason

After Nexus upload, the APK must reach end-user devices. The agreed
distribution channel is **MDM** (Mobile Device Management). The MDM
platform is not yet provisioned, so this module lands as an **echo-only
placeholder** that documents the intended download-from-Nexus ‚Üí
upload-to-MDM contract. The orchestrator wires the stage in the right
position so that when MDM is provisioned, only this module needs to
change.

## Input

- Confirm the platform team's choice of MDM (Microsoft Intune,
  VMWare Workspace ONE, Hexnode, etc.) and its API endpoint ‚Äî out of
  scope for this task. Echo placeholder until then.
- T05's contract: `result.buildVariables.NEXUS_ARTIFACT_URL` from the
  upload stage carries the URL of the freshly-uploaded APK.

## Output

### `iac/cicd-onprem/pipelines/mobile2/modules/mdm-distribute.Jenkinsfile`

A self-contained pipeline that:

- `agent { label 'Jenkins' }` ‚Äî same agent as artefact-upload, cheap
  orchestration-only work. Easy to flip to a real agent label once the
  MDM SDK / `curl` requirements are known.
- Parameters:
  - `ENVIRONMENT` (choice: bootstrap/uat/preprod/prod)
  - `ARTIFACT_TYPE` (choice: apk/aab/ipa)
  - `NEXUS_ARTIFACT_URL` (string) ‚Äî full URL of the artefact on Nexus
  - `BUILD_TAG` (string) ‚Äî for log/display only
- Single `Distribute to MDM` stage that **echoes** the intended flow:
  - "Would download artefact from Nexus: ${NEXUS_ARTIFACT_URL}"
  - "downloaded from nexus" (per user request ‚Äî explicit sentinel string
    so log greps work)
  - "Would upload to MDM (platform: TBD)"
  - "TODO: real implementation pending MDM platform provisioning"

### SIT orchestrator wiring (already done; this spec documents contract)

`mobile2/orchestrators/sit-orchestrator.Jenkinsfile`:

- New parameter `ENABLE_MDM_DISTRIBUTE` ‚Äî `booleanParam`, default `true`
  (echo step is cheap, runs by default; flip to `false` to skip).
- `Distribute to MDM` stage gated by `when { expression { params.ENABLE_MDM_DISTRIBUTE && env.UPLOAD_STATUS == 'SUCCESS' } }`.
- Runs **after** `Upload Android Artifact`, consuming
  `env.NEXUS_ARTIFACT_URL` exposed by T05.

## Acceptance criteria

- Module file exists and runs in echo-only mode.
- Stage shows the literal string `downloaded from nexus` somewhere in
  the console log (per user request).
- Skips cleanly when `ENABLE_MDM_DISTRIBUTE=false` or when the upload
  stage did not succeed.
- No actual external HTTP call attempted yet (no Nexus credential
  required by this module today).

## Notes

- When MDM is provisioned: replace the echo block with (a) a `curl
  --user $NEXUS_CRED -O ${NEXUS_ARTIFACT_URL}` and (b) the MDM-platform
  upload API call. Header docs the swap point.
- Keep `BUILD_TAG` in the param list even though it's not used today ‚Äî
  it's the canonical handle for the artefact and any MDM platform will
  want to record it as the version label.
- This is the only **net-new** module mobile2 introduces vs. legacy.
  T02-T05 are ports; T06 is greenfield.
