# T06 — Create `mdm-distribute` module (Nexus → MDM) + wire into SIT

**Status**: partial (Nexus download is real; MDM upload is still echo, blocked by MDM platform)
**Repo**: `takamulai/mithril` — `iac/cicd-onprem/pipelines/mobile2/modules/mdm-distribute.Jenkinsfile`
**Blocks**: —
**Blocked by**: T05 (MDM step consumes the Nexus URL produced by upload)

## Reason

After Nexus upload, the APK must reach end-user devices. The agreed
distribution channel is **MDM** (Mobile Device Management). The MDM
platform is not yet provisioned, so this module currently does the
**real download from Nexus** (using the shared `Nexus-cred`) and then
**echoes** the MDM-upload step. The orchestrator wires the stage in
the right position so that when MDM is provisioned, only the second
half of this module needs to change.

## Input

- Confirm the platform team's choice of MDM (Microsoft Intune,
  VMWare Workspace ONE, Hexnode, etc.) and its API endpoint — out of
  scope for this task. Echo placeholder until then.
- T05's contract: `result.buildVariables.NEXUS_ARTIFACT_URL` from the
  upload stage carries the URL of the freshly-uploaded APK.

## Output

### `iac/cicd-onprem/pipelines/mobile2/modules/mdm-distribute.Jenkinsfile`

A self-contained pipeline that:

- `agent { label 'Jenkins' }` — same agent as artefact-upload, cheap
  orchestration-only work. Easy to flip to a real agent label once the
  MDM SDK / `curl` requirements are known.
- Parameters:
  - `ENVIRONMENT` (choice: bootstrap/uat/preprod/prod)
  - `ARTIFACT_TYPE` (choice: apk/aab/ipa)
  - `NEXUS_ARTIFACT_URL` (string) — full URL of the artefact on Nexus
  - `BUILD_TAG` (string) — for log/display only
- Two stages:
  - **`Download from Nexus`** — runs a real `curl --fail --silent
    --show-error --location -u $NEXUS_USER:$NEXUS_PASS --output ...
    $NEXUS_ARTIFACT_URL`. Fails fast on HTTP >= 400. Validates the
    downloaded file is non-empty. Logs file size + SHA-256. Emits the
    literal `downloaded from nexus` sentinel after a successful pull.
    The `Nexus-cred` credential is the same one the upload module
    pushes with — Nexus authorises by repo, not direction.
  - **`Upload to MDM`** — echoes the local file path + intended MDM
    call; no actual upload until the MDM platform is provisioned.

### SIT orchestrator wiring (already done; this spec documents contract)

`mobile2/orchestrators/sit-orchestrator.Jenkinsfile`:

- New parameter `ENABLE_MDM_DISTRIBUTE` — `booleanParam`, default `true`
  (echo step is cheap, runs by default; flip to `false` to skip).
- `Distribute to MDM` stage gated by `when { expression { params.ENABLE_MDM_DISTRIBUTE && env.UPLOAD_STATUS == 'SUCCESS' } }`.
- Runs **after** `Upload Android Artifact`, consuming
  `env.NEXUS_ARTIFACT_URL` exposed by T05.

## Acceptance criteria

- Module file exists.
- Given a valid `NEXUS_ARTIFACT_URL`, the download stage pulls the
  artefact via real HTTPS to `artifacts/<sanitised-tag>.<ext>`,
  reports size + SHA-256, and prints `downloaded from nexus`.
- Given an empty `NEXUS_ARTIFACT_URL`, the stage errors out with a
  clear message ("nothing to download — upload must run first").
- Given a 404/403 (e.g. stub run before T05 lands), curl fails fast
  with the upstream HTTP error and the stage is marked FAILURE.
- `Upload to MDM` stage runs after a successful download and only
  echoes (no external HTTP call).
- Workspace cleaned in `post { always }` so APKs don't accumulate on
  the Jenkins controller.

## Notes

- When MDM is provisioned: only the `Upload to MDM` stage needs
  rewriting — the download half is already real and reusable.
- `BUILD_TAG` is used to name the local file
  (`artifacts/<sanitised-BUILD_TAG>.<ext>`). Non-alphanumeric chars
  are replaced with `_` to keep the filename portable.
- This is the only **net-new** module mobile2 introduces vs. legacy.
  T02-T05 are ports; T06 is greenfield.
- The download exercises the same `Nexus-cred` credential the
  artifact-upload module pushes with. If a stub run produces a fake
  Nexus URL (e.g. before T05 lands), the download will surface a clear
  HTTP error — that's deliberate, makes stub vs. real runs visible.
