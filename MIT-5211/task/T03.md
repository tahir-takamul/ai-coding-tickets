# T03 — Layer 15a base playbook + templates

**Status**: pending
**Repo**: `takamulai/silmarils` — `iac/eng-infra/shared-k8s/ansible/playbooks/15a-apisix.yaml` + five new templates under `iac/eng-infra/shared-k8s/ansible/templates/`
**Blocks**: T04
**Blocked by**: T01 (G2, G3, G4), T02 (image tag must exist in ACR)

## Reason

Per PLAN.md *Layer split principle*: 15a owns workload shape and
lifecycle contract (namespace, SA, daemon-config CM, paused Deployment,
ClusterIP Service); 15b owns data. 15a stays **paused** on first apply
so the Deployment never tries to start without its routes/CA bundle.
This split lets a future 15c (e.g. portal traffic class) layer on
without touching 15a or 15b.

## Input

- `playbooks/11-silmarils-gateway.yaml` — prototype for inline
  `kubernetes.core.k8s` Deployment/Service/Secret pattern.
- `playbooks/inc/preflight.yml` — reuse for tag dispatch and kubeconfig
  sanity.
- PLAN.md §15a, §Per-env variable schema, §Production hardening (probes
  + securityContext sections relevant to the Deployment).
- T01 G2 runtime UID → `securityContext.runAsUser` / `runAsGroup`.
- T02 image repository + tag from `silmarils.apisix.image.*` vars.

## Output

### `playbooks/15a-apisix.yaml`

Tagged `apisix-base`. Tasks:

1. `include_tasks: inc/preflight.yml`.
2. `kubernetes.core.k8s` apply for namespace template.
3. Apply ServiceAccount template.
4. Apply daemon-config ConfigMap template.
5. Apply Deployment template (`spec.paused: true`, `checksum/config`
   annotation placeholder = empty string on pod template).
6. Apply ClusterIP Service template.

All renders use `lookup('template', ...) | from_yaml` per the
`14d-mbridge-lfi-connector.yaml` prototype.

### `templates/apisix-namespace.yaml.j2`

- `metadata.name: {{ silmarils.apisix.namespace }}`
- `metadata.labels."pod-security.kubernetes.io/enforce": restricted`
- `metadata.labels."pod-security.kubernetes.io/enforce-version": latest`

### `templates/apisix-serviceaccount.yaml.j2`

- `metadata.name: apisix`, namespaced.
- `automountServiceAccountToken: false`.

### `templates/apisix-daemon-config.yaml.j2`

ConfigMap `apisix-daemon-config` wrapping `config.yaml` (the
daemon-level file, not routes). Key `config.yaml`. Includes
`apisix.config_provider: yaml`, `apisix.extra_lua_path`, registered
plugins (`cert-validator`, `org-router` plus the stock list silmarils
already uses), prometheus toggle, port bindings (19880 admin/debug,
19888 gateway).

Mirror the local-dev `apisix/config.yaml` byte-for-byte where possible;
env-specific concerns surface only through Jinja variables.

### `templates/apisix-deployment.yaml.j2`

- `spec.paused: true`.
- `spec.replicas: {{ silmarils.apisix.replicas }}`.
- `spec.template.metadata.annotations."checksum/config": ""` (15b
  patches it).
- `containers[0].image: {{ silmarils.apisix.image.repository }}:{{ silmarils.apisix.image.tag }}`.
- `securityContext` (pod-level): `runAsNonRoot: true`, `runAsUser`/`runAsGroup` from G2, `seccompProfile.type: RuntimeDefault`.
- `containers[0].securityContext`: `allowPrivilegeEscalation: false`,
  `readOnlyRootFilesystem: true`, `capabilities.drop: [ALL]`.
- Probes: startup + readiness TCP 19880 (failureThreshold 30, period 5s);
  liveness HTTP `/healthz` on 19888 (depends on 15b's routes CM landing
  a `public-api`-served healthz route; in T03 this is acceptable because
  the pod is paused — never runs before 15b).
- Volumes:
  - `apisix-routes` CM (subPath `apisix.yaml` → `/usr/local/apisix/conf/apisix.yaml`)
  - `apisix-org-routing` CM (subPath `org_routing.json` → `/usr/local/apisix/conf/org_routing.json`)
  - `apisix-daemon-config` CM (subPath `config.yaml` → `/usr/local/apisix/conf/config.yaml`)
  - `apisix-server-tls` Secret → mounted at the path the routes CM references
  - `apisix-ca-bundle` Secret (subPath `ca.pem` → `/usr/local/apisix/certs/ca.pem`)
  - `logs` emptyDir → `/usr/local/apisix/logs`
- `resources` from `silmarils.apisix.resources.*`.
- `automountServiceAccountToken: false`; `serviceAccountName: apisix`.
- imagePullSecrets only if T01 G1 deviation required them.

### `templates/apisix-service.yaml.j2`

- `spec.type: ClusterIP`.
- Ports: 19880 (admin/debug), 19888 (gateway).
- Selector: `app: apisix`.

## Notes

- No NetworkPolicy/PDB/HPA here — T05 adds those inside the 15b apply
  pass (after Deployment is unpaused and Ready).
- `checksum/config` annotation key MUST be on
  `spec.template.metadata.annotations`, not Deployment metadata —
  document this in the template comment so a future contributor
  doesn't "tidy" it onto the wrong object.
- The Deployment references CMs and Secrets that don't exist yet
  (`apisix-routes`, `apisix-org-routing`, `apisix-server-tls`,
  `apisix-ca-bundle`). K8s accepts this because the pod template is
  paused; volume resolution happens at pod start, which 15b triggers.
- If T01 G1 returned "managed identity does not have AcrPull", inject
  an ACR pull-secret sync task at the top of `15a-apisix.yaml`
  (mirror `07b-sync-secrets.yaml`) and reference it from the Deployment's
  `imagePullSecrets`.
