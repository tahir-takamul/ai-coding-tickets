### Smoke results (through-tunnel, from local laptop)

All three captured under `task/verified/`:
- `T08-tunnel-nocert-403.txt` — `HTTP 403`,
  body `{"error":"client certificate required"}`. cert-validator
  rejects missing header.
- `T08-tunnel-valid-fab-401.xml` — `HTTP 401`, body is a SOAP envelope
  from `dc-tang` saying no valid LFI API key. **Proves the full chain
  works**: CF Tunnel → APISIX cert-validator → org-router → forward-auth
  → dc-tang reached and responded with a structured error (not an
  infrastructure failure).
- `T08-tunnel-badcn-403.txt` — `HTTP 403`,
  body `{"error":"certificate validation failed"}`. cert-validator
  rejects unknown CN (synthetic self-signed `system@fakebank.com`).

### Findings filed during deploy

- **F08** — APISIX needs writable `/usr/local/apisix/conf`; initContainer
  pattern (`cp -rn`, NOT `cp -an`) seeds emptyDir with stock files.
- **F09** — `readOnlyRootFilesystem: true` requires per-temp-dir
  emptyDirs (multiple nginx temp dirs under `/usr/local/apisix/`).
  Disabled for first-bring-up; re-enable as a follow-up hardening.
- **F10** — `httpGet /healthz` liveness assumed a route that doesn't
  exist. Patched to `tcpSocket :19888` (same as readiness/startup).
  Template aligned with live patch.
