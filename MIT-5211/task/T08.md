### In-cluster smoke

```bash
kubectl -n $NS run smoke --rm -it --image=curlimages/curl --restart=Never -- \
  curl -k -X POST "https://apisix.$NS.svc.cluster.local:19888/api/v1/issuance-platform" \
       -H 'Content-Type: application/xml' \
       -H "X-Forwarded-Client-Cert: $(cat /tmp/valid-client.pem | jq -sRr @uri)" \
       --data @sample.xml
```
Expect 200 (or 401 from dc-tang if API key missing — both prove APISIX
→ dc-tang reached). Output → `task/verified/T08-incluster-smoke.txt`.

### Through-tunnel smoke (positive)

```bash
FQDN=silmarils-qa-apisix.takamul.cc
curl -X POST "https://$FQDN/api/commercial" \
     -H 'Content-Type: application/xml' \
     -H "X-Forwarded-Client-Cert: $(cat /tmp/valid-client.pem | jq -sRr @uri)" \
     --data @sample.xml -w "\nHTTP %{http_code}\n"
```
Expect 200. Output → `task/verified/T08-tunnel-smoke.txt`.

### Through-tunnel smoke (negative — bad CN)

Same as above with a CN not in `cn_whitelist`. Expect 403 from
`cert-validator`. Output → `task/verified/T08-badcn-403.txt`.

### Plugin load check

```bash
kubectl -n $NS logs deploy/apisix --tail=200 \
  | grep -E '(cert-validator|org-router)' \
  | tee task/verified/T08-plugins-loaded.txt
```
Both plugins should appear; no schema errors.
