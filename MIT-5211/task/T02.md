**Blocked by**: T01 (G1 for ACR pull semantics, G2 for runtime UID)

## Reason

Standalone APISIX with the silmarils custom plugins requires baking
`cert-validator.lua` (into the core plugin tree) and `org-router.lua`
(into the vendor `extra_lua_path`) into an immutable image. Plugins
versioned with the image; immutable artefact; fast pod start; no
ConfigMap-reload coupling for plugin code. Image: `silmarilsacr.azurecr.io/silmarils-apisix:3.11.0-silmarils.1`,
built via GH Actions using the existing OIDC federated identity pattern.

## Input

- `apisix/plugins/cert-validator.lua`, `apisix/plugins/org-router.lua`
  (silmarils repo, already authored).
- `apisix/config.yaml` — confirms `extra_lua_path: /opt/apisix-custom/?.lua`
  (or equivalent search-path glob).
- `.github/workflows/mbridge-mock-build-push-acr.yml` — template for
  ACR OIDC build-and-push (federated identity, no PAT).
- T01 G2 locked runtime UID.
- T01 G1 ACR-pull verdict (managed-identity AcrPull vs pull-secret sync).

## Output

### `apisix/Dockerfile`

```dockerfile
ARG APISIX_BASE=apache/apisix:3.11.0-debian
FROM ${APISIX_BASE}

ARG APISIX_VERSION
LABEL org.opencontainers.image.version="${APISIX_VERSION}" \
      org.opencontainers.image.source="https://github.com/takamulai/silmarils" \
      org.opencontainers.image.title="silmarils-apisix"

# Core plugin tree (cert-validator is invoked by name in apisix.yaml)
COPY --chown=<UID>:<GID> plugins/cert-validator.lua \
     /usr/local/apisix/apisix/plugins/cert-validator.lua

# Vendor plugin path — matches extra_lua_path in config.yaml
COPY --chown=<UID>:<GID> plugins/org-router.lua \
     /opt/apisix-custom/apisix/plugins/org-router.lua

USER <UID>
```

  pairing with the upstream image's `nonroot` (or whatever name it uses).
- The plugin file paths matter: `cert-validator.lua` must be in the core
  tree (it's named alongside built-ins), `org-router.lua` must be on the
  `extra_lua_path` glob (vendor plugin survives APISIX upgrades).
- Don't trigger the workflow as part of this task — let it fire on the
  merge. Re-run + scan locally via `build-push.sh` is acceptable

  pushed tag will look like `3.11.0-silmarils.1-0-<sha>` on a fresh tag.
  Reconcile in this task: either tag the repo with `3.11.0-silmarils.1`
  before merge, or change `variables-qa.yaml` to whatever CI emits.
