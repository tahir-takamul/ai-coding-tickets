## Reason

15a left the Deployment paused on purpose. 15b supplies the data plane:
the cert-manager-issued server cert, the CA bundle assembled from the
17 bank `.p12`s, the routes ConfigMap (rendered with cn_whitelist +
PEM material), the org-routing ConfigMap. Once all three CMs exist,
SHA256 them, patch the **pod template** `checksum/config` annotation,
and unpause — pods roll out.

## Output

### `playbooks/15b-apisix-silmarils-lfi.yaml`

Tagged `apisix-silmarils-lfi`. Tasks in strict order:

1. `include_tasks: inc/preflight.yml`.
2. Apply `apisix-server-cert.yaml.j2` (cert-manager `Certificate` CR).
3. `k8s_info` until-loop polling for Secret `apisix-server-tls` in the
   namespace, 5-minute timeout. On failure, `k8s_info` for
   `Certificate`, `CertificateRequest`, `Order`, `Challenge` and
   `debug` their status — fail loud.
4. Build CA bundle: loop over `apisix-client-certs/*.p12` with
   `community.crypto.openssl_pkcs12` (extract `other_certificates` for
   each `.p12`), concatenate PEMs into a single `ca.pem`, register the
   blob. Apply `apisix-ca-bundle` Secret from `apisix-ca-bundle.yaml.j2`.
5. Read materialized server cert+key from `apisix-server-tls` via
   `k8s_info`; base64-decode → register as `apisix_server_cert_pem` /
   `apisix_server_key_pem`. Render and apply `apisix-routes` ConfigMap
   from `apisix-routes.yaml.j2`.
6. Apply `apisix-org-routing` ConfigMap from
   `apisix-org-routing.yaml.j2` (single key `org_routing.json` =
   `silmarils.apisix.org_routing` rendered to JSON).
7. Compute SHA256 over the three CMs (`apisix-routes`,
   `apisix-org-routing`, `apisix-daemon-config`). Patch Deployment
   `apisix`:
   - `spec.template.metadata.annotations."checksum/config": <sha>`
   - `spec.paused: false`
   Single `kubernetes.core.k8s` patch, `merge_type: strategic-merge`.
8. `include_tasks: inc/rollout-wait.yml` for `Deployment/apisix`.
9. Smoke: `k8s_info` on the routes CM, `debug` print tunnel hostname
   (`silmarils.apisix.fqdn`).
