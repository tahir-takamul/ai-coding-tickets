### New file `[SIBLING] cicd/terraform/azure/saml-sonarqube.tf`

```hcl
resource "azuread_application" "sonarqube_sso" {
  display_name      = "mithril-sonarqube-sso"
  sign_in_audience  = "AzureADMyOrg"
  group_membership_claims = ["SecurityGroup"]

  identifier_uris = [
    # final value depends on application_id; templating via locals if needed
    "api://sonarqube-sso",
  ]

  web {
    redirect_uris = [
      "https://sonarqube.takamul.cc/oauth2/callback/saml",
    ]
  }

  optional_claims {
    saml2_token {
      name                  = "groups"
      additional_properties = []
      essential             = false
    }
  }
}

resource "azuread_service_principal" "sonarqube_sso" {
  application_id                = azuread_application.sonarqube_sso.application_id
  preferred_single_sign_on_mode = "saml"
  login_url                     = "https://sonarqube.takamul.cc/sessions/init/saml"
}

resource "azuread_service_principal_token_signing_certificate" "sonarqube_sso" {
  service_principal_id = azuread_service_principal.sonarqube_sso.id
  display_name         = "CN=mithril-sonarqube-sso"
}

# Assign existing group "Takamul DevOps Admin" to the SP so members can SSO in
resource "azuread_app_role_assignment" "sonarqube_sso_devops_admin" {
  app_role_id         = "00000000-0000-0000-0000-000000000000" # default app access
  principal_object_id = "0309fec6-57e6-4446-bf4f-af084823d03a"  # Takamul DevOps Admin group
  resource_object_id  = azuread_service_principal.sonarqube_sso.object_id
}
```

## Acceptance criteria

- `terraform init -upgrade` (provider re-resolution) succeeds.
- `terraform plan` shows the new application + SP + cert + role assignment, no diffs to existing Jenkins SSO (since Jenkins SSO isn't TF-managed today).
- After `terraform apply`:
  - Azure portal → Entra → Enterprise Applications: `mithril-sonarqube-sso` is present, single sign-on mode = SAML.
  - `terraform output sonarqube_sso_signing_cert_pem` returns a base64 PEM body — this is the value going into KV in T03.

## Notes

- The cert PEM from `azuread_service_principal_token_signing_certificate.value` is **base64-encoded** PFX in some provider versions and **plain PEM** in others. Verify before T03 — SonarQube wants the bare PEM body (no `BEGIN CERTIFICATE` headers) as the value of `sonar.auth.saml.certificate.secured`. Decode/strip as needed at T03 seeding time.
- `app_role_id = "00000000-0000-0000-0000-000000000000"` is the default-access app role. If `mithril-sonarqube-sso` later gets custom app roles (e.g. developer/reader for F19 follow-up), adjust this assignment to match.
- Do NOT commit the cert PEM into Git; it lives only in TF state (encrypted backend) and KV (T03).
- The `identifier_uris` value mixes `api://` with the literal `sonarqube-sso` placeholder. If a fully-templated locals form is required by the team, refactor — the smoke value works for first apply.
