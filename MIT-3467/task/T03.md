# T03 — Seed Azure Key Vault secrets

**Status**: pending
**Repo**: out-of-band (Azure portal or `az keyvault secret set` from operator workstation)
**Blocks**: T08
**Blocked by**: T02 (signing cert from TF output)

## Reason

The Ansible playbook (T06) reads SonarQube secrets from Key Vault `mithril-shared-kv` (lower subscription) via the existing `inc/fetch-and-store-secret.yaml` helper. None of these secrets exist yet. See FINDING §F10, §F11.

## Input

- KV: `mithril-shared-kv` (subscription `7047239e-8819-4015-9344-5a7481a9439b`, lower).
- Naming convention: `cicd-sonarqube-<key>` (matches existing `cicd-*` prefix in `cicd_secret_names`).
- The signing cert from T02's `terraform output sonarqube_sso_signing_cert_pem` (sensitive).

## Output

Six secrets created in `mithril-shared-kv`:

| Secret name | Value |
|---|---|
| `cicd-sonarqube-admin-password` | `openssl rand -base64 24` (target admin pwd, random) |
| `cicd-sonarqube-current-admin-password` | literal string `admin` (chart rotates to the target on first sync — F10) |
| `cicd-sonarqube-db-password` | `openssl rand -base64 24` (used for both Postgres admin and `sonarUser`) |
| `cicd-sonarqube-monitoring-passcode` | `openssl rand -base64 24` |
| `cicd-sonarqube-saml-cert` | x509 PEM body from T02's `terraform output -raw sonarqube_sso_signing_cert_pem`, **stripped of `BEGIN CERTIFICATE` / `END CERTIFICATE` headers** (Sonar expects the raw body) |
| `cicd-sonarqube-token` | placeholder (e.g. literal `placeholder` or empty) — filled in T10 after first login |

### Setting via CLI

```
az keyvault secret set --vault-name mithril-shared-kv --name cicd-sonarqube-admin-password --value "$(openssl rand -base64 24)"
az keyvault secret set --vault-name mithril-shared-kv --name cicd-sonarqube-current-admin-password --value "admin"
az keyvault secret set --vault-name mithril-shared-kv --name cicd-sonarqube-db-password --value "$(openssl rand -base64 24)"
az keyvault secret set --vault-name mithril-shared-kv --name cicd-sonarqube-monitoring-passcode --value "$(openssl rand -base64 24)"

# SAML cert — strip headers if present
SAML_CERT=$(terraform output -raw sonarqube_sso_signing_cert_pem | sed -e '/-----BEGIN CERTIFICATE-----/d' -e '/-----END CERTIFICATE-----/d' | tr -d '\n')
az keyvault secret set --vault-name mithril-shared-kv --name cicd-sonarqube-saml-cert --value "$SAML_CERT"

# Placeholder token — replaced in T10
az keyvault secret set --vault-name mithril-shared-kv --name cicd-sonarqube-token --value "placeholder"
```

## Acceptance criteria

- `az keyvault secret list --vault-name mithril-shared-kv --query "[?starts_with(name,'cicd-sonarqube-')].name" -o tsv` returns all six names.
- The MI from T01 (`mithril-mi-sonarqube`) can read at least one of these — verify by `az keyvault secret show -n cicd-sonarqube-admin-password --vault-name mithril-shared-kv` from the operator's account first; runtime read by the SonarQube pod is verified at T08.

## Notes

- T02 must have been applied already; the cert PEM comes from there.
- Once the chart has rotated `admin` to the target password (after T08 first sync), keep `cicd-sonarqube-current-admin-password` updated to whatever the live admin pwd is — required for any subsequent re-runs of the playbook to authenticate the rotation.
- Consider tagging all six secrets with `service=sonarqube,owner=cicd` so a future inventory sweep can find them.
- Do NOT commit any of these values into Git, even temporarily.
