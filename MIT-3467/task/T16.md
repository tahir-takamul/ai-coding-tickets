### `.github/actions/sonar-pr-report/action.yml`

Composite action with these inputs (required unless noted):
`component`, `pr-number`, `commit-sha`, `cf-access-id`, `cf-access-secret`, `jenkins-user`, `jenkins-token`, `sonar-token`, `github-token`, `jenkins-url` (default `https://jenkins.takamul.cc`), `sonar-url` (default `https://sonarqube.takamul.cc`), `max-inline-comments` (default `20`).

Logic (recommend Node 20 with `@actions/core` + `@actions/github` + native `fetch`; bash + `jq` + `gh` is also fine):

1. **Poll Jenkins** ‚Äî exponential backoff 10s ‚Üí 60s. Capture build URL + result (`SUCCESS`/`FAILURE`/`UNSTABLE`).
2. **If Jenkins failed**: post failing GitHub Check; exit non-zero.
3. **Fetch Sonar data** ‚Äî three GETs with `CF-Access-Client-Id/Secret` + `Authorization: Bearer ${sonar-token}`:
   - `qualitygates/project_status?projectKey=mithril-${component}`
   - `measures/component?component=mithril-${component}&metricKeys=bugs,vulnerabilities,code_smells,security_hotspots,coverage,duplicated_lines_density,ncloc`
   - `issues/search?componentKeys=mithril-${component}&resolved=false&ps=500`
4. **Fetch PR diff** ‚Äî `gh api /repos/${owner}/${repo}/pulls/${pr-number}/files --paginate`. Note added/modified line ranges per file.
5. **Filter issues** ‚Äî keep only issues whose `(component, line)` intersects an added/modified line in the PR diff. Sort by severity (BLOCKER > CRITICAL > MAJOR > MINOR > INFO) ‚Üí top `max-inline-comments`.
6. **Upsert GitHub Check** (idempotency ‚Äî F23):
   - `GET /repos/{owner}/{repo}/commits/${commit-sha}/check-runs?check_name=SonarQube+%2F+${component}` ‚Äî find existing.
   - If found: `PATCH /repos/{owner}/{repo}/check-runs/{id}`.
   - Else: `POST /repos/{owner}/{repo}/check-runs`.
   - `name: "SonarQube / ${component}"`, `status: completed`, `conclusion: success|failure|neutral`, `output.title`, `output.summary`.
7. **Upsert PR summary comment** ‚Äî find by marker `<!-- sonar-${component} -->`; PATCH or POST. Body: metrics table + dashboard link + failing-conditions list.
8. **Post PR review comments** for filtered issues ‚Äî `POST /repos/{owner}/{repo}/pulls/${pr-number}/reviews` with `event: COMMENT` and `comments[]`. Each: `path`, `line`, `side: RIGHT`, `body` with severity + rule + Sonar issue link.

### `SONARQUBE_READ_TOKEN` (GitHub repo secret)

Generate a Sonar **user token** scoped to a read-only `github-actions` user (Browse perm on the three projects). Add to GitHub repo Settings ‚Üí Secrets ‚Üí Actions as `SONARQUBE_READ_TOKEN`.

### Sourcing of phase 1 signal
GHA never reads Sonar's quality-gate verdict directly for the pass/fail decision. The chain is: Sonar gate verdict ‚Üí `sonar.qualitygate.wait=true` makes the gradle/scanner exit non-zero on FAIL ‚Üí Jenkins build inherits FAILURE ‚Üí wrapper sonar-pr inherits via `wait+propagate` ‚Üí GHA polls and reads `.result` ‚Üí Check conclusion.
