## Reason

The SonarQube pod must read secrets (admin pwd, DB pwd, monitoring passcode, SAML cert) from Azure Key Vault `mithril-shared-kv`. The auth model is **AKS workload identity** — pod surfaces an OIDC token, exchanged for a managed-identity token that has KV `Get,List` perms. No static credentials.

This mirrors the Jenkins MI block at `cicd/terraform/azure/identity.tf:9-73`. See FINDING §F08.

## Input

- Read `[SIBLING] cicd/terraform/azure/identity.tf` lines 9-73 — the Jenkins MI + federated cred + KV access policy template. Mirror this shape.
- Read `[SIBLING] cicd/terraform/azure/outputs.tf` — to add the new client_id output alongside the existing Jenkins one.
- Confirm the AKS OIDC issuer URL data source is already declared in the module (Jenkins block uses it). Reuse, don't redeclare.
