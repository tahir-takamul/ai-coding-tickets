## Reason

Prove the rollout works end-to-end before declaring done: pod readiness, Service DNS, CF Access, SAML login, scanner-in-cicd-agent, Jenkins-credential-to-Sonar-token round-trip. A trivial smoke pipeline exercises the full path.

## Input

- The new `cicd-agent` image tag from T07 is in ACR.
- Jenkins agent pod template tag is bumped to the new image (this task makes the bump if T07 did not).

## Output

### 1. Bump Jenkins agent image tag

Find where the `cicd-agent` image tag is referenced in the Jenkins agent pod template — typically in `[SIBLING] cicd/k8s/jenkins/values.yaml` (search for `cicd-agent`) or in pipeline pod-template Groovy. Bump to the T07 tag, commit, and re-run `01-jenkins.yml` (or the equivalent values-only sync the team uses).

### 2. Health checks (foreground)

```
kubectl -n cicd get sts sonarqube-sonarqube -o jsonpath='{.status.readyReplicas}'    # → 1
kubectl -n cicd exec sonarqube-sonarqube-0 -- curl -s localhost:9000/api/system/status   # → {"status":"UP"}
kubectl -n cicd get sa sonarqube -o jsonpath='{.metadata.annotations.azure\.workload\.identity/client-id}'   # matches MI client_id
kubectl -n cicd get pvc | grep sonar                                                  # 3 PVCs Bound on managed-csi-premium
```

### 3. Cloudflare gate

```
curl -I https://sonarqube.takamul.cc                                                  # → 302 to CF Access challenge
curl -H "CF-Access-Client-Id: ..." -H "CF-Access-Client-Secret: ..." https://sonarqube.takamul.cc/api/system/status   # → {"status":"UP"}
```

### 4. SAML login (browser)

- Open `https://sonarqube.takamul.cc` in a fresh browser session.
- CF Access challenges; complete CF SSO.
- On Sonar login page click **Log in with Azure Entra ID**.
- Should land logged in as DevOps Admin user.

### 5. Smoke pipeline (Jenkins)

In Jenkins, create a one-off pipeline job (or a temp Jenkinsfile in a feature branch under `[SIBLING] cicd/pipelines/`) with:

```groovy
pipeline {
  agent { kubernetes { yaml '''<existing cicd-agent pod template at the new tag>''' } }
  stages {
    stage('Sonar smoke') {
      steps {
        withCredentials([string(credentialsId: 'sonar-token', variable: 'SONAR_TOKEN')]) {
          sh '''
            sonar-scanner --version
            mkdir -p /tmp/smoke && cd /tmp/smoke
            echo "console.log('hello');" > index.js
            sonar-scanner \
              -Dsonar.host.url=http://sonarqube-sonarqube.cicd.svc:9000 \
              -Dsonar.login=$SONAR_TOKEN \
              -Dsonar.projectKey=smoke-test \
              -Dsonar.sources=.
          '''
        }
      }
    }
  }
}
```

Replace `'sonar-token'` with the actual credential ID Jenkins assigned to the KV `cicd-sonarqube-token` secret (read from Manage Jenkins → Credentials).

### 6. Verify in Sonar UI

- Log in to `https://sonarqube.takamul.cc`.
- **Projects** view should show `smoke-test` with at least one issue (the `console.log` hits the no-console rule).
- Click in — analysis date matches the smoke pipeline run.

## Acceptance criteria

- All four health checks (step 2) pass.
- CF gate behavior (step 3) matches the two expected responses.
- Browser SAML flow (step 4) lands logged-in as DevOps Admin.
- Smoke pipeline (step 5) runs to green.
- `smoke-test` project visible in Sonar UI with issues.

## Notes

- After the smoke test, **delete** the `smoke-test` project from Sonar (Administration → Projects → Management → Delete) so it doesn't sit on the dashboard. Or leave it as evidence — operator's call.
- If the smoke pipeline can't find the `sonar-token` credential, T10 step 5 didn't fully wire it into Jenkins. Re-run `01-jenkins.yml` with the secrets tag.
- If the smoke pipeline gets `Connection refused` on `sonarqube-sonarqube.cicd.svc:9000`: the agent pod is not in the `cicd` namespace. Confirm the agent pod template's namespace matches; in same-namespace AKS, no FQDN suffix is needed.
- If the smoke pipeline gets `401 Unauthorized` from Sonar with the right token: token may be expired or scoped wrong. Check Sonar UI → Account → Security → token list for the `jenkins-scanner` token's "Last used" timestamp.
- DO NOT use a real production project name for the smoke (e.g. don't call it `mithril-backend`). The per-module project keys are reserved for the per-module follow-up.
